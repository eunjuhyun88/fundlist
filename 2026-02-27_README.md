# fundlist

Investment data collection agent that fetches from multiple sources, stores in SQLite, and returns a list view.

## What It Collects

- `openclaw` via GitHub API (`Virtual-Protocol/openclaw-acp`)
  - releases
  - issues
  - commits
- `sec` via SEC EDGAR
  - recent filings by ticker
- `fred` via FRED API
  - latest macro values by series id
- `coingecko` via CoinGecko API
  - latest market snapshots for selected coins

## VC Ops Assistant (실시간 갱신/기록/데드라인 알림)

```bash

  --output /Users/ej/Downloads/vc-list/fundlist/data/reports/vc_ops_report.md

# 2) 날짜 기준 제출 리스트
python3 fundlist.py ops-list --from-days -365 --to-days 30 --limit 60

# 3) Speedrun만 확인
python3 fundlist.py ops-list --speedrun-only --include-no-deadline --limit 40

# 4) 특정 프로그램 제출 리포트 (예: Alliance DAO)

# 5) 감시 루프 (예: 15분마다 20회)
```

## Submission Discovery Agent (Apply/Pitch 자동 수집)

`speedrun.a16z.com` 같은 공식 제출 폼 페이지를 자동으로 발견하고, 정규화해서 DB + 리포트로 저장한다.

```bash
# 1) 기본 수집: 검색 + 기존 fundraising DB seed를 함께 사용
python3 fundlist.py submission-scan \
  --max-sites 40 \
  --max-pages-per-site 4 \
  --max-results-per-query 6 \
  --http-timeout 8 \
  --output /Users/ej/Downloads/vc-list/fundlist/data/reports/submission_targets_report.md

# 2) 섹터/스테이지/지역 포커스 스캔
python3 fundlist.py submission-scan \
  --sector "web3,ai" \
  --stage "pre-seed,seed" \
  --region "us,apac" \
  --min-score 5 \
  --json-output /Users/ej/Downloads/vc-list/fundlist/data/reports/submission_targets.json

# 3) 특정 URL seed만 빠르게 스캔
python3 fundlist.py submission-scan \
  --skip-search \
  --no-fundraise-seeds \
  --seed-urls 'https://speedrun.a16z.com,https://renewvc.com/apply'

# 4) 저장된 결과 조회
python3 fundlist.py submission-list --limit 50 --min-score 4

# 5) 리포트만 재생성
python3 fundlist.py submission-report \
  --limit 100 \
  --min-score 4 \
  --output /Users/ej/Downloads/vc-list/fundlist/data/reports/submission_targets_report.md

# 6) JSON 스키마 포함 export
python3 fundlist.py submission-export \
  --limit 200 \
  --min-score 4 \
  --output /Users/ej/Downloads/vc-list/fundlist/data/reports/submission_targets.json
```

저장 필드(핵심):
- `org_name`, `org_type`
- `org_type` 값: `VC` / `Accelerator` / `Grant` / `Angel syndicate` / `Unknown`
- `submission_url`, `submission_type` (`form` / `email` / `intro-only`)
- `status` (`open` / `rolling` / `deadline` / `closed`)
- `requirements`, `notes`, `source_page_snapshot`, `evidence`, `score`
- 변경 감시: `submission_target_events` 테이블에 생성/수정 diff 기록

## OpenClaw Multi-Agent (ACP)

`openclaw-acp`를 이용해 여러 에이전트에 동시에 작업을 생성하고, 상태를 폴링해 통합 보고서를 만든다.

사전 준비:

```bash
# 1) openclaw-acp 설치 및 setup
git clone https://github.com/Virtual-Protocol/openclaw-acp /path/to/openclaw-acp
cd /path/to/openclaw-acp
npm install
npm link
acp setup
```

실행:

```bash
cd /Users/ej/Downloads/vc-list/fundlist
export OPENCLAW_ACP_DIR="/path/to/openclaw-acp"
python3 fundlist.py openclaw-multi \
  --query "fundraising investor outreach" \
  --max-agents 3 \
  --requirements-json '{"task":"introduce relevant VCs for AI/Web3 startup"}' \
  --output "/Users/ej/Downloads/vc-list/fundlist/data/reports/openclaw_multi_agent_report.md"
```

드라이런(실제 job 생성 없이 후보만 확인):

```bash
python3 fundlist.py openclaw-multi --query "fundraising" --dry-run
```

요청값을 offering별로 다르게 주려면:

```json
{
  "generate_image": { "prompt": "AI startup fundraising infographic", "ratio": "16:9" },
  "0xWalletAddress|donation_20": { "amount": 20, "symbol": "USDC" }
}
```

```bash
python3 fundlist.py openclaw-multi \
  --query "fundraising" \
  --requirements-map /absolute/path/requirements_map.json
```

한 채널(그룹)에서 마케팅봇과 같이 운영:

```bash
# 그룹 충돌 방지(기본값 1): 그룹에서는 @봇명 명령만 처리
export TELEGRAM_REQUIRE_MENTION_IN_GROUP=1
```

그룹에서 명령 예시:

```text
/fundraise@vcreechew_bot
/fundraise_ai@vcreechew_bot gemini
/openclaw_dry@vcreechew_bot fundraising outreach
/submit_report@vcreechew_bot alliance dao
```

명령이 아닌 일반 텍스트를 보내면, `TELEGRAM_CHAT_MODE=1`일 때 AI 대화 응답을 준다.

대화형 응답(지금 Codex와 대화하듯이) 사용:

```bash
# 기본값 1: 일반 텍스트에도 AI 응답
export TELEGRAM_CHAT_MODE=1

# provider 선택: auto|groq|gemini|huggingface|openrouter (기본 auto)
export TELEGRAM_CHAT_AI_PROVIDER=auto
```

대화형 예시:

```text
@vcreechew_bot 오늘 Web3 VC 중에서 바로 컨택할 5곳 뽑아줘
@vcreechew_bot 이 데이터로 1주 실행계획 짜줘
```

핵심:
- 같은 그룹에 봇 2개를 추가해도 됨.
- 각 봇은 `@봇명`이 붙은 명령만 처리하게 운영 권장.
- 이 설정이면 마케팅봇 명령을 `vcreechew_bot`이 가로채지 않음.

주의:

- 토큰은 코드에 저장하지 말고 환경변수로만 사용.
- 공개 채팅에 토큰이 노출되었으면 즉시 BotFather에서 재발급/회수.

## Environment Variables

- `FUNDLIST_DB`: custom sqlite path
- `OPENCLAW_REPO`: default `Virtual-Protocol/openclaw-acp`
- `SEC_TICKERS`: default `AAPL,MSFT,NVDA,TSLA,AMZN`
- `SEC_USER_AGENT`: SEC user-agent string
- `FRED_API_KEY`: required for FRED connector
- `FRED_SERIES`: default `FEDFUNDS,CPIAUCSL,UNRATE,GDP`
- `COIN_IDS`: default `bitcoin,ethereum,solana`
- `COINGECKO_API_KEY`: optional for CoinGecko
- `GITHUB_TOKEN`: optional for GitHub rate limits
- `FUNDRAISE_FILES`: fundraising source file paths (comma-separated)
- `FUNDRAISE_REPORT_PATH`: default report output path
- `GROQ_API_KEY`: Groq API key for AI summary
- `GROQ_MODEL`: default `llama-3.3-70b-versatile`
- `GEMINI_API_KEY`: Gemini API key for AI summary
- `GEMINI_MODEL`: default `gemini-2.0-flash`
- `HF_TOKEN`: Hugging Face token (or `HUGGINGFACE_API_KEY`)
- `HUGGINGFACE_MODEL`: default `Qwen/Qwen2.5-7B-Instruct`
- `HUGGINGFACE_BASE_URL`: default `https://router.huggingface.co/v1/chat/completions`
- `OPENROUTER_API_KEY`: OpenRouter API key
- `OPENROUTER_MODEL`: default `openai/gpt-4o-mini`
- `OPENROUTER_BASE_URL`: default `https://openrouter.ai/api/v1/chat/completions`
- `AI_PROVIDER`: default AI provider (`groq` or `gemini` or `huggingface` or `openrouter`)
- `AI_MODEL`: default shared model flag value
- `OPENCLAW_ACP_DIR`: local openclaw-acp repo path
- `OPENCLAW_ACP_CMD`: explicit ACP command (example: `acp`)
- `OPENCLAW_MULTI_REPORT_PATH`: default openclaw multi-agent report output path
- `TELEGRAM_BOT_TOKEN`: Telegram bot token
- `TELEGRAM_ALLOWED_CHATS`: comma-separated allowed chat IDs
- `TELEGRAM_REPORT_CHAT_ID`: auto report target chat/channel ID
- `TELEGRAM_REQUIRE_MENTION_IN_GROUP`: default `1` (handle only `/cmd@this_bot` in group chats)
- `TELEGRAM_CHAT_MODE`: default `1` (enable free-text AI replies)
- `TELEGRAM_CHAT_AI_PROVIDER`: `auto|groq|gemini|huggingface|openrouter` (default `auto`)
- `TELEGRAM_CHAT_GROQ_MODEL`: optional chat model override for Groq
- `TELEGRAM_CHAT_GEMINI_MODEL`: optional chat model override for Gemini
- `TELEGRAM_CHAT_HUGGINGFACE_MODEL`: optional chat model override for Hugging Face
- `TELEGRAM_CHAT_OPENROUTER_MODEL`: optional chat model override for OpenRouter
- `TELEGRAM_CHAT_HISTORY_TURNS`: in-memory context turns per chat (default `6`)
- `TELEGRAM_CHAT_SYSTEM_PROMPT`: optional system prompt override for chat mode
- `VC_OPS_REPORT_PATH`: default VC ops report output path
- `VC_PROGRAM_REPORT_PATH`: default single-program submission report path
- `VC_OPS_PROGRAMS`: cron auto-report target keywords (comma-separated, default `alliance dao`)
- `VC_OPS_PUSH_TELEGRAM`: `1` to auto-push cron reports to Telegram (`0` to disable)
- `VC_OPS_ALERT_DAYS`: default deadline alert window for ops commands/bot
- `OPENCLAW_BOT_MAX_AGENTS`: telegram `/openclaw_run` default max agents
- `OPENCLAW_BOT_TIMEOUT`: telegram `/openclaw_run` timeout seconds
- `OPENCLAW_BOT_POLL_INTERVAL`: telegram `/openclaw_run` poll seconds
